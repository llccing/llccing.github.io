## 前言

作为一个爱折腾的程序员，我一直在寻找一个可以本地部署的 AI 助手解决方案。从最初的「价格肉疼」到现在的「真香警告」，我的 OpenClaw 探索之路充满了踩坑与惊喜。

## 第一部分：初试家用 PC

最开始的想法很简单：让 OpenClaw 帮我整理 OneDrive 中的截图和重复图片。

### API 费用爆炸

当时给 OpenClaw 配的是 OpenCode 的 API key，用的是 Claude Code 4.5（现在都 4.6 了）。结果随便问一个简单问题，比如「你是什么模型」，就要消耗近 **1 美元**！

> 这谁扛得住？一个简单对话就就没了一杯奶茶钱。

### 意外收获：生成小说封面

不过在家用 PC 上还是做出了一些成果的。我让它帮我生成小说的封面图，调用 **nano banana** 模型，效果居然还不错！

## 第二部分：转战云端

### 为什么选腾讯云？

家里电脑没有公网访问能力是个硬伤。恰好最近了解到：

- **订阅模式**不会价格爆炸
- **腾讯云**服务稳定，性价比高
- **咸鱼上的 Mini Max 2.5**，30 块钱月订阅，价格感人

于是开始跃跃欲试！

### 当前进度

目前的双线作战：

1. **家里 PC**：继续跑图生成任务，API 成本问题已解决
2. **云上龙虾**：给了 GitHub 权限，开始通过聊天的方式记录踩坑过程

> 整个过程速度快很多，真的很有意思！

## 第三部分：一周实战记录（3月17日-23日）

部署到腾讯云后，我开始在不同渠道测试 OpenClaw 的能力。这一周的探索让我对 AI 助手有了全新的认识。

### 📱 QQ 渠道（3月17日-19日）

**主要成就：创作《至尊宝在上海》小说系列**

- **3月17日**：首次通过 QQ 与 OpenClaw 对话，测试基本功能
- **3月18日-19日**：开启小说创作项目
  - 规划了 10 篇大话西游风格的现代短篇小说
  - 每篇 1500-2000 字，主角是在上海打拼的至尊宝
  - 完成了 3 轮编辑打磨：
    - 第 1 轮：削减重复句式，丰富紫霞人格
    - 第 2 轮：增加感官描写和内心独白
    - 第 3 轮：文笔打磨，优化对话节奏
  - 在 GitHub 创建了 10 个 issues 记录每章内容
  - 最终完成约 26,000 字的完整系列

**技术亮点：**
- 通过 QQ 发送语音消息进行交互
- 使用 `<qqimg>` 标签发送图片
- 使用 `<qqfile>` 标签发送 PDF 简历
- 配置了定时任务（每日英语阅读、早晨提醒）

### 💬 LightClawBot 渠道（3月19日-23日）

**主要成就：博客自动化与内容管理**

**3月19日：每日英语阅读自动化**
- 配置凌晨 4:00 定时任务，自动生成 B2 级英语文章
- 解决了时区问题（UTC → Asia/Shanghai）
- 配置腾讯云 COS 存储音频文件
- 实现文章自动发布到博客

**3月20日：博客迁移与修复**
- 迁移旧博客文章到新框架
- 修复文章 404 问题
- 配置 10 分钟进度汇报提醒
- 完成小说系列的 pubDatetime 修正

**3月22日：GitHub 仓库分析**
- 分析 98 个 GitHub 仓库
- 生成「10年前端考古」博客文章
- 记录从 2016 到 2026 的技术成长轨迹

**3月23日：仓库清理与个人品牌**
- 归档 57 个旧仓库（从 98 个精简到 43 个）
- 更新 GitHub Profile README
- 更新个人简历（添加敏捷开发、AI 辅助开发经验）
- 处理 PR review comments

### 🔧 技术配置优化

**模型分配策略：**
- 主会话：`ctok/claude-opus-4-6`（高质量任务）
- 子代理：`qwencode/qwen3.5-plus`（节省成本）
- ctok 额度：$40/天，$160/周
- qwencode：基本无限额

**定时任务配置：**
- 每日英语阅读：凌晨 4:00（lightclawbot）
- 早晨提醒：早上 7:00（qqbot）
- oTree 论坛扫描：每 4 小时（优化后）

**文件存储方案：**
- 腾讯云 COS：`reading-audios-1308187607` bucket
- 公开访问配置，无需登录认证
- 用于存储音频、PDF 等静态资源

## 总结

| 方案     | 成本         | 优点     | 缺点       |
| -------- | ------------ | -------- | ---------- |
| API 按量 | ~$1/简单问答 | 无需维护 | 费用不可控 |
| 订阅模式 | 30元/月      | 成本可控 | 需要云服务 |

从「太贵了」到「真香」，OpenClaw 确实给我带来了全新的 AI 助手体验。

### 关键收获

1. **多渠道协作**：QQ 用于创作，LightClawBot 用于技术任务，各有所长
2. **自动化流程**：定时任务 + GitHub 集成，实现内容自动化生产
3. **成本优化**：合理分配模型，主会话用 Opus，子任务用 Qwen
4. **实战价值**：不只是聊天工具，而是真正的生产力助手
