# 国泰航空业务知识库

## 航空业务基础

### 核心业务流程
- **订票系统 (Booking)**: 航班搜索、座位选择、价格计算
- **值机系统 (Check-in)**: 在线/机场值机、座位分配、登机牌
- **常旅客计划 (FFP - Frequent Flyer Program)**: 里程累积、会员等级、兑换
- **航班管理**: 航班状态、延误通知、行李追踪

### 关键术语
- **PNR (Passenger Name Record)**: 旅客订座记录
- **GDS (Global Distribution System)**: 全球分销系统（Amadeus, Sabre, Galileo）
- **IATA**: 国际航空运输协会
- **Codeshare**: 代码共享航班
- **Interline**: 联程航班

## 前端技术挑战

### 性能要求
- **高并发**: 促销期间大量用户同时访问
- **实时性**: 航班状态、座位库存实时更新
- **多语言**: 中英文切换，繁简体支持
- **跨时区**: 全球用户，时区处理复杂

### 业务复杂度
- **复杂表单**: 多乘客、多航段预订
- **支付集成**: 多种支付方式、货币转换
- **规则引擎**: 票价规则、退改签政策
- **无障碍**: WCAG 2.1 AA 标准合规

---

## 技术建议

### 架构层面
1. **模块化设计**: 按业务域拆分（订票、值机、会员）
2. **状态管理**: 使用 NgRx/Signals 管理复杂状态
3. **缓存策略**: 航班数据、用户信息分层缓存
4. **错误处理**: 网络异常、超时的优雅降级

### 性能优化
1. **懒加载**: 按路由拆分 bundle
2. **虚拟滚动**: 航班列表、座位图
3. **图片优化**: WebP、懒加载、CDN
4. **预加载**: 关键路径资源预加载

### 工程化
1. **组件库**: 统一 UI 组件，提升开发效率
2. **E2E 测试**: 关键流程自动化测试
3. **性能监控**: Real User Monitoring (RUM)
4. **A/B 测试**: 功能灰度发布
